CloudFormation Creation

We’ll create a custom CloudFormation template so you can run any application on Amazon Web Services. With this CloudFormation Template, you can create multiple environments. For example, you can create one for Production with many nodes, plus a few smaller environments for development, QA, or performance testing.

CloudFormation Template Features:

  • VPC
  • Public and private subnets
  • 3 availability zones
  • Security Groups
  • Can be used in any AWS region

Dockerize an Application

We will help you transform your application* into a Docker container so that it can be easily deployed out to the CoreOS nodes created by the CloudFormation template.

Automated Deployment

We will provide you with a set of tools (and documentation) so you can easily deploy code into your environments.

Automated Deployment Tools:

  • Container building, tagging, and pushing
  • Deployment of containers into an environment
  • Rolling updates deployment
  • Local development setup

Logging

Viewing logs is critical to running and troubleshooting a cloud application. We offer a few options for central log aggregation with a web interface for easy log management.

Features:

  • Multi user access
  • Searching/querying
  • Alert creation based on searches
  • Building dashboards

Log Vendor Options:

  • Loggly.com
  • LogEntries.com
  • GrayLog

Server Monitoring

Gain visibility into your entire server environment

We monitor your servers using New Relic Server Monitoring (a free service). This provides metrics on:

  • CPU usage
  • Memory usage
  • Hard disk usage
  • Server processes
  • Docker containers
