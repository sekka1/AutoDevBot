AutoDevBot makes it easy for you to monitor all of your API endpoints.

Are all of your API endpoints working properly?

Applications are getting more complex with internal and external API interactions between various servers and processes. The response from your APIs represent your application stack. Keeping track of what is acceptable behavior is paramount to keeping a healthy system going.

ReUse Code In Development

The same Frisb.js code that you use to define the monitoring test for AutoDevBot, you can use that in development also. You simple run the same code locally with Node.js. This makes it easy for your developers and the operations group to know what will be monitored.

Know when something goes wrong

Get notified immediately when a test does not return what you expect. Get detailed test messages and drill down into where the problem occurred. We also provide additional callbacks and integrations into your workflows.

How Does It Work?

We are making it dead simple to use so that you can get up and running in matter of minutes.

  1. We will create a GitHub.com repository for you and share it with you.
  2. This repo already has working example monitors that you can start with.
  3. Every commit that you make will automatically deploy and be used for monitoring.

AutoDevBot will automatically pull this GitHub repo in and start monitoring. The only other thing left for you to do is to provide an email or a callback URL to AutoDevBot so that it can send notifications when there is an failure event.
